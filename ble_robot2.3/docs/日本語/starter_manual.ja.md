# スターターマニュアル (初回導入)

この文書は、初学者または初回セットアップ担当者が、最短で「接続して動かす」まで到達するための手順書です。配布版運用を前提に、Python 開発経験がなくても進められる構成にしています。

## 1. 事前準備

- Windows PC
- ロボット本体 (BLE 通信可能)
- 配布一式 (exe, cf.rbcc, meta, tools, docs)

## 2. Thonny 導入 (ロボット側確認を行う場合)

ロボット側プログラムの書き込み・確認が必要な授業では Thonny を利用します。

1. Thonny をインストールする
2. Interpreter を授業で指定された環境に合わせる
3. ロボット側との接続設定を確認する
4. 最小の送受信テストを実施してから PC アプリへ進む

Thonny は必須ではありませんが、ロボット側要因の切り分けに有効です。

## 3. PC アプリ初回起動

1. ble_robot_2.3.exe を起動
2. 歯車ボタンで BLE アドレスを設定
3. Connect を押して接続
4. 十字キーまたはゲームパッドで応答確認

## 4. 設定ファイルの理解

- cf.rbcc: 利用者設定 (接続先、操作キー、UI)

この分離により、配布後の変更は rbci 側で吸収でき、exe の再編集が不要になります。

## 5. 初回トラブル対応

接続できない場合は、BLE アドレス、ロボット電源、接続待機状態を順に確認してください。設定反映に問題がある場合は、実行フォルダに cf.rbcc と meta フォルダが存在し、書き込み権限があるかを確認します。
