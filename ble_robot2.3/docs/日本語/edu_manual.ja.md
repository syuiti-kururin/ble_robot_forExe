# EDUマニュアル (授業実施者向け)

教育利用では、機能説明よりも「失敗しにくい進行」を優先します。ここでは授業運用で実際に使える進行テンプレートを示します。

## 授業設計の原則

- 安全確認を先に行う
- 1コマの到達目標を1つに絞る
- 成功体験を先に作る
- 最後に原理を言語化する

## 推奨進行 (45-90分)

1. 目標共有: 今日は何ができるようになるか
2. 実演: 接続から1動作まで
3. 体験: 受講者が同手順を実施
4. 調整: 接続不良を一緒に切り分け
5. ふりかえり: 設定保存と再現確認

## ファイル管理の教え方

受講者には cf.rbcc を「自分の設定ファイル」として説明します。

## 授業後の再現性

授業後は cf.rbcc と meta のバックアップを残し、次回授業開始前に差分確認を行います。これが複数クラス運用の安定化に有効です。
