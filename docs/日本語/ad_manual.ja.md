# アドミンマニュアル (公開運用範囲)

この文書は、配布・公開・運用管理を担当する管理者向けです。ここでは公開してよい範囲に限定して、構成と運用方針を明示します。

## 公開版で説明する責務分離

- cf.rbcc: 利用者設定の正本
- meta/*.rbci: 内部運用データ
- meta/*.rbcl: ロゴ等の関連データ

データ分割の目的は、配布後に保守変更を安全に行うことと、実行ファイル本体への直接編集を前提にしないことです。この方針が崩れると、保守コストと事故率が上がります。

## 推奨配布構成

- ble_robot_2.3.exe
- cf.rbcc
- meta/
- tools/
- docs/

tools は内部設定の管理作業で使用します。利用者向け配布では不要な場合がありますが、運用保守担当には同梱を推奨します。
なお、一般配布者向けにはそもそも同梱して公開されていないものもあります。

## 変更管理ポリシー

- ユーザー体験に関わる設定は cf.rbcc
- 運用・内部仕様に関わる設定は rbci
- 変更履歴は配布単位で記録し、復旧用バックアップを保持

## 公開時チェック

1. 起動と接続
2. 設定保存
3. rbci 変更反映
4. tools 起動
5. ドキュメント整合
