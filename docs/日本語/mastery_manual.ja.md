# マスタリーマニュアル (保守・高度運用)

この文書は、障害調査・保守更新・運用監査を担当する上級者向けです。日常運用で見落としやすい検証観点をまとめます。

## 設定ファイル体系

- cf.rbcc: ユーザー設定の正本
- meta/inc.rbci: ロゴ関連の内部設定
- meta/inh.rbci: 内部版情報
- meta/indef.rbci: デフォルト上書き
- meta/inabout.rbci, meta/inlicense.rbci: 表示文面

## 整合チェックの基本

1. 起動時に cf.rbcc 読み込みが成功する
2. About/License が rbci の内容を表示する
3. rbci 変更が再読み込み後に反映する
4. tools から同一データにアクセスできる

## 障害時の切り分け

- 接続系: BLE 層と設定層を分離して確認
- 保存系: 権限、配置、排他を確認
- 表示系: 対象 rbci の更新有無を確認

## 保守運用の推奨

変更前後で cf.rbcc と meta をセットでバックアップし、配布単位で管理します。単一ファイルだけの差し替えは追跡不能の原因になるため避けてください。
